# Image Compression Coursework

Курсовой проект по сжатию изображений на языке C.

Тема проекта: реализация и сравнение методов сжатия изображений:

- SVD-сжатие;
- JPEG-подобное сжатие на основе DCT и квантования.

Также в проекте реализуются метрики качества PSNR и SSIM для сравнения исходного и восстановленного изображений.

## Что уже реализовано

На текущем этапе реализовано:

- начальная структура проекта;
- сборка через `Makefile`;
- обработка аргументов командной строки;
- чтение 24-bit BMP-файлов;
- сохранение BMP-файлов;
- хранение изображения в структуре `Image`;
- преобразование изображения в матрицу яркости;
- преобразование изображения в оттенки серого;
- расчёт MSE;
- расчёт PSNR;
- разбиение изображения на блоки 8x8;
- прямое DCT для блока 8x8;
- обратное DCT для блока 8x8;
- квантование JPEG-коэффициентов;
- обратное квантование;
- JPEG-подобная обработка всего изображения;
- параметр силы квантования для JPEG-режима.

## Запуск

Общий формат запуска зависит от режима работы программы.

### Grayscale

```bash
./image_compressor grayscale images/test_beck.bmp output.bmp
```

В этом режиме программа:

1. загружает BMP-изображение;
2. переводит его в оттенки серого;
3. считает PSNR между оригиналом и grayscale-версией;
4. сохраняет результат в BMP-файл.

### Blocks

```bash
./image_compressor blocks images/test_beck.bmp output.bmp
```

В этом режиме программа:

1. загружает изображение;
2. переводит его в матрицу яркости;
3. проверяет возможность деления на блоки 8x8;
4. выводит информацию о количестве блоков;
5. показывает первый блок, DCT-коэффициенты, квантованные коэффициенты и восстановленный блок.

Этот режим используется для отладки JPEG-алгоритма.

### JPEG-like mode

```bash
./image_compressor jpeg images/test_beck.bmp output.bmp 1.0
```

Последний параметр — коэффициент квантования.

Примеры:

```bash
./image_compressor jpeg images/test_beck.bmp output_q1.bmp 1.0
./image_compressor jpeg images/test_beck.bmp output_q5.bmp 5.0
./image_compressor jpeg images/test_beck.bmp output_q10.bmp 10.0
```

Чем больше коэффициент, тем сильнее квантование и тем больше потери качества.

Примерная логика:

```text
1.0  - стандартное квантование
5.0  - более сильные потери
10.0 - заметное ухудшение качества
```

## Ограничения текущей версии

Сейчас программа поддерживает только:

- BMP-файлы без сжатия;
- изображения с глубиной цвета 24 бита;
- изображения, размеры которых делятся на 8 для JPEG-режима;
- JPEG-подобное восстановление в BMP-файл, а не сохранение настоящего `.jpg`.

Важно: результат JPEG-режима сохраняется как BMP, поэтому размер выходного файла пока не показывает реальную степень сжатия. Сжатие оценивается через потери качества, PSNR и дальнейшую статистику DCT-коэффициентов.
